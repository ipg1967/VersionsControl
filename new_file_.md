# Инструкция по GIT

> *Все инструкции написаны кровью*

## 1. Установка GIT на Mac

Открываем Терминал и пишем:

*Если установлен Homebrew :*  
`brew install git`

*Если нет, то вводим эту команду:*
`git --version`

После этого появится окно, где предложат установить **Command Line Tools (CLT)**
Соглашаемся и ждем установки. Вместе с CLT установится и **git**

Еще вариант - с сайта https://git-scm.com/downloads - установка последней версии GIT

## 2. Настройка GIT 
<!--- Это делается один раз (так пишутся невидимые комменты) --->

Открываем терминал MacOS и вводим следующие команды:

*Установим имя пользователя*
*Вместо <ваше_имя> можно ввести, например, Vasya_Pupkin.  Кавычки оставляем*:

`git config --global user.name "<ваше_имя>"`

*Теперь установим email. Принцип тот же.*

`git config --global user.email "<адрес_почты@email.com>"`

## 3. Инициализация репозитория 

1. Создадим репозиторий (отслеживаемую папку). Перейдите в папку вашего проекта. Для MacOS путь может выглядеть как */Users/UserName/Desktop/MyProject*
Введите команду в терминале:

`cd <путь_к_вашему_проекту>`

2. Инициализация/создание репозитория 

`git init`

**Git** начал отслеживать изменения файлов в папке. Запишем изменения файла. Для этого необходимо создать commit (сохранение).

3. Чтобы добавить все файлы проекта нужна команда commit (именно с точкой):

`git add .`

Или так

`git add --all`

Если хотим добавить конкретный файл то можно так

`git add <имя_файла> `

4. Теперь создаем commit. 
![природа команды>](</Users/igorbenko/Desktop/coding_files/commit.jpg>)
Обязательно указываем комментарий. В кавычках!

`git commit -m "<комментарий>"`

5. Чтобы посмотреть текущее состояние ветки, например, какие файлы добавлены или не добавлены для создания commit, можно выполнить команду:

`git status`

***OK***

> Не стоит после каждого изменения файла делать commit. Чаще всего их создают, когда:
> * Создан новый функционал
> *	Добавлен новый блок на верстке
> *	Исправлены ошибки по коду
> *	Вы завершили рабочий день и хотите сохранить код

> Это поможет держать ваши ветки в чистоте и порядке. Тем самым, вы будете видеть историю изменений по каждому нововведению в вашем проекте, а не по каждому файлу.

## 4. Просмотр истории коммитов  
 
 `git log` - вывести все изменения в папке. Тут же можно скопировать от 4 и более знаков кода коммита, чтобы перейти на него (см. п.5).
 
 `git log --oneline` - вывести все изменения коротко, только с комментами

## 5. Перемещение между сохранениями  

Переход на нужное сохранение:

`git checkout код нужного сохранения`

Возврат на основную ветку:

`git checkout master`
 

## ДРУГИЕ КОМАНДЫ

`git rm` - удалить файл

`git diff` - ппосмотреть разницу с последней версией

`git switch` - - это возврат назад из предыдущих сохранений

*Подсказки по популярным командам:*

`git help`

*По конкретной команде:*

`git help <название_команды>`



Чтобы в терминале вернуться вкомандную строку - нажать клавишу ***q***
